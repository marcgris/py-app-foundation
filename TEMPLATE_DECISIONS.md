# Architecture and Design Decisions (Template)

This file should be copied to docs/plan/DECISIONS.md in every new project created from this template.

## How to Use
- Add a new entry for each major decision (technology, structure, workflow, etc.)
- Reference the date and rationale for each decision

## Example Entries

### 2026-04-11: Use FastAPI for API overlay
- Rationale: Async support, OpenAPI docs, strong ecosystem

### 2026-04-11: Require .copilot-instructions.md in all projects
- Rationale: Ensures Copilot/agent guardrails and workflow clarity

### 2026-04-11: Place agent/skill files in .github/skills and .github/instructions
- Rationale: Enables Copilot/agent discovery and maintainability
