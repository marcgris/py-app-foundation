# Contributing Guide (Template)

This file should be copied to docs/guide/CONTRIBUTING.md in every new project created from this template.

## Workflow
- Read ROADMAP.md and DECISIONS.md for project context
- Claim or create an issue before starting work
- Create a feature branch for your changes
- Run all validation commands before opening a PR
- Follow code style and documentation standards

## Code Standards
- Type hints everywhere (pyright strict)
- Google-style docstrings
- Organized imports (ruff)
- 80%+ test coverage

## For Copilot/Agents
- Follow .copilot-instructions.md for guardrails and workflow
- Do not modify planning or workflow docs without approval
